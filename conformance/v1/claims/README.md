# Claims

Each implementation claim should declare:

- implementation name
- implementation version
- claimed protocol version
- claimed binding versions
- claimed compliance profiles
- optional surfaces not implemented

Claims should be narrower than or equal to the verified result.

Seed artifact:

- `claim-template.md`
- `core-http-claim-seed.md`
- `core-tools-http-claim-seed.md`

First seeded target:

- `Core` over `HTTP Binding v1`

Next seeded target:

- `Core + Tools` over `HTTP Binding v1`
